# Android APK via TWA (Trusted Web Activity)

**Date:** 2026-05-07  
**Status:** Approved

## Goal

Produce an installable `.apk` for Android tablets that runs the existing Globos Fulfilment web app full-screen, without browser chrome, and without touching the main frontend codebase or disrupting Vercel deploys.

## Approach: TWA via Bubblewrap

A Trusted Web Activity wraps the live Vercel URL inside a thin native APK. The web app runs in a Chrome process — same engine, same capabilities — but displayed as a standalone app (no address bar). Since the warehouse always has internet, offline capability is not required.

**Chosen over Capacitor because:**
- Java JDK + Android Studio (~12 GB) not installed and not justified for an always-online app
- Bubblewrap only needs Java JDK (~200 MB download)
- No APK rebuild needed for app updates — Vercel deploys are immediately live
- The app already has a compliant PWA manifest and service worker

## Repository Layout

```
Globos fulfilment Software/
├── frontend/                        ← unchanged React + Vite app
│   └── public/
│       └── .well-known/
│           └── assetlinks.json      ← new: one static file, no effect on app
└── android-twa/                     ← new: Bubblewrap project, gitignored build output
    ├── twa-manifest.json
    ├── android/
    └── ...
```

## Components

### 1. Java JDK 17
Installed via Homebrew. Required by Bubblewrap to invoke Gradle and Android build tools.

### 2. Bubblewrap CLI
`@bubblewrap/cli` installed globally via npm. Handles Android SDK download, project scaffolding, signing key generation, and APK build.

### 3. `android-twa/` project
Initialised with `bubblewrap init` pointing at:
- Manifest URL: `https://<vercel-domain>/manifest.json` — confirm exact domain in Vercel dashboard before running `bubblewrap init`
- Package name: `com.globos.fulfilment`
- App name: `Globos Fulfilment`
- Icons sourced from existing `/icon-192.png` and `/icon-512.png`

### 4. `assetlinks.json`
Generated by Bubblewrap after signing key creation. Added to `frontend/public/.well-known/assetlinks.json` and deployed to Vercel. This file lets Android verify the APK is authorised to open the domain without showing a browser bar. It is a static JSON file — zero runtime impact on the app.

### 5. Signed APK
`bubblewrap build` produces `app-release-signed.apk`. A keystore (`globos.keystore`) is created and stored in `android-twa/` — must be kept safe for future builds.

## Data Flow

```
Tablet (APK)
  └─ opens Chrome TWA process
       └─ loads https://<vercel-domain>
            └─ React app → Railway backend (HTTPS)
                         → Supabase (realtime)
```

No API URL changes required — the app already uses the Railway production URL when deployed to Vercel.

## Update Flow

1. Fix bug / add feature in `frontend/`
2. Push to `main` → Vercel deploys automatically
3. Tablet sees new version on next app open — **no new APK needed**

A new APK build is only needed when changing: app name, icons, splash screen, or package name.

## Out of Scope

- Offline / cached asset support (Capacitor would be needed for this)
- Play Store publishing
- Push notifications (separate project if needed)
- Multiple signing environments (debug vs release)

## Prerequisites Checklist

- [ ] Homebrew installed on dev machine
- [ ] `brew install openjdk@17` completed
- [ ] `npm install -g @bubblewrap/cli` completed
- [ ] Exact Vercel domain confirmed via Vercel dashboard (needed for `assetlinks.json` and `bubblewrap init`)
- [ ] Keystore file backed up after first build

## Risks

| Risk | Mitigation |
|------|-----------|
| Keystore lost → can't update APK | Back up `globos.keystore` + password to secure location |
| `assetlinks.json` not reachable → browser bar shown | Verify via `https://<domain>/.well-known/assetlinks.json` after deploy |
| Bubblewrap downloads Android SDK on first run (~500 MB) | Normal, happens once |
